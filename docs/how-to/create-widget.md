# Как разработать новый виджет после рефакторинга

В проекте обновлена архитектура виджетов, чтобы сделать их создание проще для всех членов команды, независимо от уровня владения классами. Теперь каждый виджет состоит из двух частей:

1. **Класс-обёртка** – отвечает за интеграцию с системой (загрузку данных, управление жизненным циклом, события `Ready`/`Complete`). Написан на классах, реализует `WidgetComponent` и наследуется от `BaseComponent`.  
2. **Функциональный модуль** – содержит чистые функции для построения DOM и обработки взаимодействий пользователя. Использует только стандартный DOM API, без классов.

Теперь можно не мучатся с классами, а вместо этого сосредоточиться на функциональном модуле. 

---

## 1. Общий контекст виджета

Для передачи данных из класса-обёртки в функциональный модуль используется тип `WidgetContext` (находится в `src/types/widget-types.ts`). Он содержит:

- `widgetId: string` – идентификатор виджета, необходим для отправки ответов;
- `onReady: () => void` – колбэк, который нужно вызвать, когда виджет полностью загружен и готов к отображению;
- `onComplete: () => void` – колбэк, который нужно вызвать, когда пользователь успешно завершил задание.

В функциональном модуле вы будете получать этот контекст и использовать его колбэки.

---

## 2. Структура функционального модуля

Создайте файл `src/features/your-widget/your-widget-functional.ts`. В этом файле:

- Определите вспомогательные функции для построения DOM (например, `createQuestion`, `createTrueButton`, `createFalseButton` и т.д.). Они создают элементы, устанавливают классы и текстовое содержимое.
- Экспортируйте основную функцию `renderYourWidget`, которая принимает данные виджета и контекст, строит DOM и возвращает объект с полем `element` (корневой элемент) и методом `destroy` (для очистки).

**Основные шаги внутри функции рендера:**

1. Создайте корневой контейнер `container` (обычный `div`).
2. Объявите переменные для хранения выбранного ответа, ссылки на кнопку отправки и т.п.
3. Напишите функцию `buildUI`, которая:
   - очищает `container.innerHTML`,
   - создаёт все необходимые элементы (вопрос, варианты ответов, кнопку отправки),
   - навешивает обработчики событий (выбор ответа, отправка),
   - собирает элементы в структуру (например, карточку) и добавляет в `container`.
4. Вызовите `buildUI()`.
5. Вызовите `context.onReady()`.
6. Верните объект `{ element: container, destroy: () => container.remove() }`.

**Важно:**
- При правильном ответе (после получения вердикта от `widgetDataSource.submitAnswer`) вызовите `context.onComplete()`.
- Обрабатывайте возможные ошибки (например, через `alert`).
- Все вспомогательные функции выносите для ясности, но можно писать код прямо в `buildUI`.

---

## 3. Класс-обёртка (уже существует)

Для каждого типа виджета есть класс-обёртка, реализующий `WidgetComponent`. Например, для true‑false вы можете создать `TrueFalseWidgetCreator` (по аналогии с `QuizWidgetCreator`). Он:

- Загружает данные виджета через `widgetDataSource`.
- Создаёт контекст и вызывает функциональный рендер.
- Вставляет полученный элемент в свой контейнер.
- Прокидывает события `Ready` и `Complete`.

**Вы можете скопировать класс-обёртку из существующего виджета и адаптировать.** Минимальные изменения: заменить имя класса, тип виджета, импорты.

---

## 4. Добавление виджета в фабрику

Фабрика виджетов (`WidgetFactory`, файл `src/core/widget-factory.ts`) отвечает за создание экземпляра виджета по типу. Добавьте новый тип в фабрику, импортировав свой класс-обёртку и добавив ветку в `switch` с созданием экземпляра.

---

## 5. Использование виджета в TicketPageController

`TicketPageController` теперь принимает массив заданий с указанием типа и ID. Вы можете использовать его для последовательного прохождения нескольких виджетов (например, билета). Пример на странице:

- Импортируйте `TicketPageController`.
- В функции страницы создайте массив `items` с объектами `{ type: 'your-widget', id: '...' }`.
- Создайте экземпляр `TicketPageController` с этим массивом и верните его из `render()`.

Можно смешивать разные типы виджетов в одном билете:
```typescript
const items = [
  { type: 'quiz', id: 'quiz-1' },
  { type: 'true-false', id: 'tf-001' },
  { type: 'memory-game', id: 'gc-001' },
];
```

---

## 6. Важные замечания

- **Стили**: размещайте CSS для виджета в отдельном файле `*.scss` и импортируйте его в функциональном модуле (или в классе-обёртке). Рекомендуем импортировать в функциональном модуле, чтобы стили были привязаны к самому виджету.
- **Интернационализация (i18n)**: если в виджете есть текст, который должен переводиться, используйте `language$` и `translations`. В функциональном модуле можно подписаться на изменения языка и обновлять текст. Для простоты можно пока оставить текст на английском, а локализацию добавить позже.
- **Обработка ошибок**: в функциональном модуле используйте `alert` или другой способ уведомления об ошибках. В будущем мы заменим на единую систему уведомлений.
- **Документация**: для каждого виджета создайте файл `types.ts` с описанием структур данных (виджет, ответ, ошибки).

---

## 7. Что нужно сделать, чтобы добавить свой виджет?

1. Создать папку `src/features/your-widget/`.
2. Определить типы в `types.ts`.
3. Написать функциональный модуль `your-widget-functional.ts` (следуя описанной структуре).
4. Создать класс-обёртку `YourWidgetCreator.ts` (скопируйте из существующего виджета и адаптируйте).
5. Добавить тип виджета в константы `widgetTypes` и в тип `WidgetType`.
6. Добавить создатель в `WidgetFactory`.
7. Создать страницу (если нужна отдельная страница) или использовать `TicketPageController` в существующей.
